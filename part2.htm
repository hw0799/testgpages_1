2.Introduction and Description of Data:Description of relevant knowledge. Why is this problem important? Why is it challenging? Introduce the motivations for the project question and how that question was defined through preliminary EDA.

 

More specifically, Biomarker data is the focus of these project. Longitudinal nature of the data is considered and emphasis is more on data preprocessing. Based on the nature of this problem, several multi-classification model is evaluated in order to identify the model with good performance and identify the predictors that are most effective predictors with respective to this problem with given data set.

Two dataset were considered: ADNIMERGE.csv and BIOMARK.csv; ADNIMERGE.csv is the main dataset, and step by step when refine data pre-processing part, at final stage, BIOMARK.csv is used. BIOMARKER predictors were selected in a way that still could get valid dataset after join with ADNIMERG.csv in order to perform model evaluation. Dataset has following issue or characteristics:

 

a.       Limited patient data available, especially after removing rows with missing data

This the concern, inside this project, the model built on the data perform well on test set. But to move further, imputing missing data to get large data set, build and test on the larger data set is encouraged and should be one of the next steps.

 

b.      Data is collected according to multiple protocols (ADNI, ADNI GO, ADNI2), there is ADNI 3, but only have less than 300 records, not enough to build credible model, so didnt be part of the analysis in this project.

When split data for model building and testing, multiple protocols is considered, and data is split by protocol ADNI1, ADNI2, ADNI GO. Train/test data is split within ADNI1 first, model were built on ADNI1 train data, then tested on ADNI1 test, ADNI 2, ADNI GO.

 

c.       Some of the data has longitudinal components: one patient could have multiple visit with multiple records (entry), some information is fixed, for example, demographic, like gender, education, ethic, race, etc.

 

d.      Because one of the major work is data preprocessing, this will impact all the following step/results: model building/evaluation etc. So each step has its own model and results corresponding to its own pre-processed data.

 

e.       Data pre-processing approach:

Step 1. One participant could have multiple entry, DX is the diagnosis results, and could change over time for one patient. There are longitudinal data issue. To solve this, DX_last is created to keep track one participants last available diagnosis results. DX_last is the response variable. Models were built to predict DX_last.

Step 2: based on step 1, DX_change is created to keep track if a participants diagnosis results changes over time. If it is positive, the participants records will be dropped from dataset. DX_last is the response variable, this step remove noise that might impact the prediction power of the model, due to longitudinal nature of the data

Step 3: based on step 2, predictors from BIOMARK.csv are selected, then join with ADNIMERGE.csv based on participant ID, exam date, and protocol.

Step 3 also break into two parts: Step3_a: CSF biomarker were tested and step3_b: Plasma biomarker were also tested. The enhancement of adding additional biomarker predicters were assessed.
